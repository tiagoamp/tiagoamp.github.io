Project SABI-WEB

DESCRIPTION:
Project goal was to re-implement the front-end and business logic layers of a critical application, making sure of no side-effects from the previous PL/SQL version.
The legacy system was causing memory overflow in the database server due to non-separated layers.
DURATION:
10 months (Start date: 02/Jan/2008 - End date: 10/Oct/2008).
ROLE:
Developer / Team Leader.
RESPONSIBILITIES:
Java implementation; Solution design; Project status report.
TECH & TOOLS:
Java, EJB, JDBC, JSP, Struts, Oracle Database, PL/SQL.
ACHIEVEMENTS:
Project hit its goal before the deadline (about 30 days earlier). Application rebuilt with scalable architecture and running in JavaEE application servers. Solved database server memory overflow problem, with final account measure about 20% of free memory.

Project SAE/PREVAGENDA

DESCRIPTION:
Project goal was to provide online scheduling for public government services requests.
The system should create a virtual queue and schedule the citizens' attendance in government agencies nearest to their homes.
There was a previous unstable version of the application developed in PL/SQL, and the team should rewrite/refactor the code segregating presentation and business logic in Java, providing an architecture that supports the large number of online requests from all over the country.
DURATION:
03 months (Start date: 24/Sep/2007 - End date: 10/Dez/2007).
ROLE:
Developer / Data Administrator.
RESPONSIBILITIES:
Implementation in Java and PL/SQL; Data modeling; Documentation of features and design.
TECH & TOOLS:
Java, EJB, JDBC, JSP, Struts, Oracle Database, PL/SQL.
ACHIEVEMENTS:
Increase efficiency in public services, solving real life blocking/awaiting work problem. Built robust application with skippable downtime.
