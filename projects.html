Project SIBE-REQUER

DESCRIPTION:
Project goal was to build an application to manage the lifecycle of requests for public services (assistential and retirement) in government agencies.
When our team was assigned to this project, it was in progress and several months delayed. Stakeholders was displeased. Requirements was gathered and fixed in contract some years before, and changes wasn't allowed. Codebase was over engineered. There was no shippable product after more than one year of project.
Project was cancelled after negociation between our dev team, PMO and stakeholders, since its purpose was outdated. We convinced PMO to use an MVP approach to a new version of the project.
DURATION:
05 months (Start date: 03/Aug/2009 - End date: 31/Dec/2009).
ROLE:
Developer / Team Leader
RESPONSIBILITIES:
Java implementation; Project status report; Scope management.
TECH & TOOLS:
Java, EJB, JSF, JPA, Oracle Database.
ACHIEVEMENTS:
Negotiate with PMO, clients and key stakeholders to abort the project and change to an MVP (Minimum Viable Product) approach. No more efforts and budget spent with outdated requirements, saving money and time of development (about $7000 savings per month in dev effort).

Project SAE/PREVAGENDA

DESCRIPTION:
Project goal was to provide online scheduling for public government services requests.
The system should create a virtual queue and schedule the citizens' attendance in government agencies nearest to their homes.
There was a previous unstable version of the application developed in PL/SQL, and the team should rewrite/refactor the code segregating presentation and business logic in Java, providing an architecture that supports the large number of online requests from all over the country.
DURATION:
03 months (Start date: 24/Sep/2007 - End date: 10/Dez/2007).
ROLE:
Developer / Data Administrator.
RESPONSIBILITIES:
Implementation in Java and PL/SQL; Data modeling; Documentation of features and design.
TECH & TOOLS:
Java, EJB, JDBC, JSP, Struts, Oracle Database, PL/SQL.
ACHIEVEMENTS:
Increase efficiency in public services, solving real life blocking/awaiting work problem. Built robust application with skippable downtime.
